# CLAUDE.md

## 言語

日本語を使う。

## コミット前の作業

git commit の前に以下を必ず行う。

### 仕様書（`docs/spec.md`）

変更内容に応じて仕様書を作成・更新する。

### 作業日報（`docs/diary/YYYY-MM-DD.md`）

その日の作業内容・判断・残課題を記録する。

### ADR（`docs/adr/NNNN-タイトル.md`）

技術選定・アーキテクチャ判断など将来の意思決定に影響する決定をした場合に作成・更新する。

### 引き継ぎ情報（`docs/handover.md`）

次の担当者が迷わず作業を引き継げるよう、現状・未完了タスク・注意点を更新する。
内容が大きくなる場合は `docs/plan/` 配下に作業計画ファイルを分割して管理する。

### 関連ドキュメントの更新

変更の影響を受けるドキュメントを更新し、各ファイルの先頭に最終更新日を記載する。

```
最終更新: YYYY-MM-DD
```
