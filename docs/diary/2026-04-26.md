# 作業日報 2026-04-26

## 作業内容

Remix + Cloudflare Workers + D1 を使った住所録アプリケーションの初期実装を行った。

### 実施した作業

1. **プロジェクト構成の設計**
   - Remix v2 + `@remix-run/cloudflare-workers` + Vite の構成を選択
   - Drizzle ORM で D1 への型安全なアクセスを実現
   - Tailwind CSS でシンプルな UI を実装

2. **認証機能の実装**
   - Workers Secrets（AUTH_USERNAME, AUTH_PASSWORD）による認証
   - Cookie セッション管理（createCookieSessionStorage）
   - タイミング攻撃対策（HMAC ベースの定数時間比較）
   - CSRF 対策（Synchronizer Token Pattern）

3. **住所録 CRUD の実装**
   - 一覧・新規追加・詳細・編集・削除の全ルート
   - Zod によるバリデーション
   - キーワード検索・ページネーション

4. **セキュリティ対策**
   - CSP ヘッダー付与
   - robots noindex
   - UUID 主キー（IDOR 対策）
   - POST-Redirect-GET パターン

### 技術的判断

- **認証方式**: 外部認証（Auth0 等）を使わず Workers Secrets のみで実装
  → 無料プラン内で完結させるため
- **セッション**: KV ではなく Cookie セッションを選択
  → KV の書き込み上限（1,000回/日）を消費しないため
- **パスワードハッシュ**: bcrypt/argon2 ではなく平文 Secret + HMAC 比較
  → Workers の WASM 制限で bcrypt が使えない場合を考慮。Secret 自体は暗号化保存される

5. **テスト・lint・CI の追加**
   - Vitest + Testing Library で単体テスト実装
   - ESLint 9（フラット設定）で静的解析
   - GitHub Actions CI ワークフロー（lint → typecheck → test の順）

   テスト対象（Workers 固有の API に依存しない層をターゲット）:
   - `validation.ts` - Zod スキーマの網羅テスト（境界値・正常/異常系）
   - `auth.server.ts` - タイミングセーフ認証の動作確認（Node 20 の crypto で動作）
   - `csrf.server.ts` - CSRF トークン生成・検証（session をモック化）
   - `ContactForm.tsx` - フォーム構造・アクセシビリティ属性の確認

   テストしない対象と理由:
   - ルートファイル（loader/action）: D1 バインディングが必要で Workers 環境が要る
   - Workers エントリーポイント: 統合テストの範囲

### 技術的判断（追加分）

- **テストランナー**: Jest ではなく Vitest を選択
  → Vite ベースのプロジェクトで設定を共有でき、ESM ネイティブで Workers に近い
- **DOM 環境**: jsdom ではなく happy-dom を選択
  → より軽量・高速。React Testing Library と互換性あり
- **Worker バインディングのテスト**: 今回は対象外
  → `@cloudflare/vitest-pool-workers` が別途必要で複雑。単体テストで十分なカバレッジを確保

### デプロイエラーの修正（2段階）

デプロイ時に以下のエラーが連続して発生した。

**第1エラー**（初回）: `virtual:remix/server-build` の解決失敗
```
✘ [ERROR] Could not resolve "virtual:remix/server-build"
    workers/app.ts:24:23
```
- 原因: wrangler が `workers/app.ts`（ソース）を直接 esbuild でバンドルしようとした。`virtual:remix/server-build` は Vite の仮想モジュールで wrangler からは解決できない。
- 暫定対処: `main = "build/server/index.js"` に変更。

**第2エラー**（暫定対処後）: Module Worker 形式エラー + Workers Sites 非対応
```
▲ [WARNING] build/server/index.js has exports like an ES Module, but hasn't defined a default export
✘ [ERROR] Workers Sites does not support uploading versions through `wrangler versions upload`
```
- 原因1: `build/server/index.js` は Remix のサーバーバンドル（named exports）であり、Cloudflare Worker エントリポイントの `export default { fetch() }` を持たない。
- 原因2: `[site]`（Workers Sites）は `wrangler versions upload` 非対応。`[assets]` へ移行が必要。

**最終修正**:

1. `scripts/build-worker.js` を作成: esbuild で `workers/app.ts` をバンドルし `build/worker/index.js` を生成。`virtual:remix/server-build` を `build/server/index.js` にエイリアス解決する。
2. `package.json` の `build` スクリプト: `remix vite:build && node scripts/build-worker.js`
3. `wrangler.toml` の更新:

| 設定 | 変更前 | 変更後 |
|------|--------|--------|
| `main` | `workers/app.ts` → `build/server/index.js` | `build/worker/index.js` |
| 静的アセット | `[site] bucket = "./public"` → `./build/client` | `[assets] directory = "./build/client"` |

ビルドフロー:
1. `remix vite:build` → `build/server/index.js`（Remix アプリ）+ `build/client/`（静的アセット）
2. `node scripts/build-worker.js` → `build/worker/index.js`（完全な Worker バンドル、`export default { fetch() }` 付き）
3. wrangler がそのままデプロイ（再バンドル不要）

## テーマカラー切り替え機能の追加（追記）

### 変更内容

- `app/app.css`: CSS カスタムプロパティ（`--color-primary-500/600/700`）を追加。`[data-theme="green"]` でグリーンテーマを定義。`.link-primary`・`.text-primary` コンポーネントクラスを追加
- `app/components/ThemeToggle.tsx`: 新規作成。セレクトボックスでテーマを切り替えるコンポーネント
- `app/root.tsx`: フラッシュ防止インラインスクリプトを `<head>` に追加。ErrorBoundary の `text-indigo-600` を `text-primary` に変更
- `app/routes/contacts._index.tsx`: ヘッダーに `ThemeToggle` を配置
- `app/routes/contacts.$id.tsx`: `text-indigo-600 hover:underline` を `link-primary` クラスに変更

### 技術的判断

- **既存 @apply はそのまま**: `@layer components` の外側に `[data-theme="green"]` ルールを書くことで、specificity を問わず優先させた。既存コードの変更を最小限に抑えるため
- **`localStorage` + インラインスクリプト**: SSR のフラッシュを防ぐため `<head>` の `<Links>` より前にインラインスクリプトを置き、ハイドレーション前にテーマ属性を設定

## 気づき・注意点

- D1 は `updated_at` の自動更新トリガーをサポートしないため、
  アプリ側で明示的に `new Date().toISOString()` を設定している
- Remix の routes ファイル名で `$id_.edit` の `_` は重要。
  省略すると `$id` の子ルートになり Outlet が必要になる
- `.dev.vars` を `.gitignore` に必ず含めること（認証情報漏洩防止）
